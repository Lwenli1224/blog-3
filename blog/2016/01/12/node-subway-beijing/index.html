Node.js

让我们谈谈「生产环境中的 Node.js」- Node 地下铁第 1 次线下沙龙总结

作者: 淘杰 发表于: 2016-01-12

1 月 9 日下午，Node 地下铁第 1 次线下沙龙在北京京仪大酒店举行，当天也是难得的好天气。

本次沙龙邀请到了 4 位线上 Node.js 大规模应用设计以及性能调优的核心工程师，共同就「生产环境中的 Node.js」这一主题进行分享。

感谢各位同学的参与，13 点半签到，很快会议厅就被挤满了，对那些全场站在后排听完分享的同学表示抱歉～

分享

Node.js 助力淘宝大促活动

峰值 4 万 QPS 的大促会场催生了百亿成交量的全球狂欢节。支撑着这些玲琅满目的会场和频道页面的，正是一套基于 Node.js 的页面搭建、编辑和发布的全方位解决方案。

还记得知乎上的那个帖子吗，Node.js 在双十一中有哪些应用，表现如何？相信你可以在竹暄同学的分享中找到答案。

微服务之路

Docker 加上 Node.js，将大型的、复杂的、长期运行的应用程序构建为一组相互配合的服务，每个服务都可以很容易得到局部改良。

2014 年互联网比较热的两个技术热点，一个是容器虚拟化技术Docker，一个是微服务架构（MicroServices Architecture）。什么鬼，我都不知道，2014 年我在干嘛啊。。。 还好有@张挺 挺爷的分享

PM2.5 美团的 Node.js 线上运维和监控

PM2.5 是一套基于 PM2 开发的开源监控报警系统，可以方便的进行线上 Node.js 集群的实时监控和管理。

也许您的应用还在使用 PM2，不妨听听美团@郭凯同学的分享，尝试下 PM2.5，说不定是一个更好的选择呢。


数亿个文字的旅程

用户在石墨文档中敲出的每一个字都经过了服务器的层层中转。这一个个文字的完美旅程，正是要归功于石墨对 Node.js 的贯彻应用与对自动化测试、部署和监控的深入实践。

应用崩溃后，除了自动重启，我们还能做些什么？除了万能的 console.log，还有其他的选择吗？看看石墨@炉音同学有啥绝招。


讨论

4 位同学精彩的分享结束后，就参会同学之前报名时提交的一些问题，进行了一场讨论。各位嘉宾也分享了各自在生产环境中使用 Node.js 遇到的一些线上问题。

  • 线上故障分享，如何尽快发现异常，恢复
  • 生产环境中的 Node.js 升级
  • 微服务的开源社区解决方案
  • 应用服务器的 QPS 如何评估等等

不知道同学们还有没有印象呢， 不妨看一遍视频回顾一下吧～

结束语

再次感谢各位同学的参与，我们年后再约！也非常感谢博文视点对本次活动的图书赞助！
